# TempFly_ER Future
A rework is planned, and there is much to cover to keep you informed. We will go over almost everything here and include a FAQ afterwards for any other questions you may have.

## The Code & Name
It's old news to say that most of TF's code is in need of being re-written. Much is written with older Java Lang that was deprecated a long time ago. This created a few issues while working on adding to TF_ER with the new features this branch has. So this is a necessary step in the continuation of TF_ER. As such, I've decided to make this full rework as it's own separate plugin under a new name. This will most likely be "RevFly", unless a better name comes up between now and the deadline for the rework's completion. However, just because it's planned to be reworked and renamed does NOT mean it's going to be completely different from the plugin you know and love! RevFly will still have the same features and extensive customization, just with the benefits of an updated code base and shiny new name!

## The Roadmap
Now, we can't just right in the deep end just yet. There are things to consider before this rework will begin and patches to be made in the meantime. For starters, we will be working to answer these two very critical questions:
1. What version of Java will be the target?
2. Based on #1, what versions of Minecraft will be supported?

This will be done with the help of bStats and feedback from server owners or managers like you. These two things are critically needed as it determines the root Java code base used and what new or legacy supports are needed.

## FAQ
**Whem will the rework begin?**
> ~~Sometime in May 2026.~~ Rework has begun and a release date will be announced as soon as one is determined.

**What is the current target versions?**
> As of May 25th, Minecraft 1.20.x and Java 21. This is currently based on only data from [bStats](https://bstats.org/plugin/bukkit/TempFly_XCraft/29194)

**Will TempFly_ER continue to recieve updates until XFly is complete?**
> Only bug fix updates. Feature changes, additions, etc. will still be accepted, but will likely be put on hold until rework is complete. Please keep reporting bugs as you find them and sharing your ideas.

**Will TempFly_ER recieve an update for Minecraft 26.x?**
> Patch released on June 10, 2026 for TempFly v3.2.0 is available from the v3.2.0 release page. Version 26.x should work as intended with this patch, but be sure to report any bugs you find so they may be squished.

**How can I let you know if I'm using TempFly_ER on an earlier version of Minecraft and/or Java?**
> If using v3.2.0, this should automatically be reported to [bStats](https://bstats.org/plugin/bukkit/TempFly_XCraft/29194) unless you've disabled it. If you have, or are using an earlier version of TempFly_ER (v3.1.8-v.1.11), please jump into [my Discord server](https://discord.gg/jP2uyYzbCA) to let me know. (Make sure you select "Plugins" from Onboarding questions or "Channels & Roles" to see TempFly channels).

**What about previously planned feature additions?**
> Database to database migration is delayed until RevFly (the complete rework) is completed.

**Can I help patch issues with TempFly_ER?**
> Sure! I will still be watching for PR's and if it fixes an issue, I'll release the patch!

**Can I help with the rework?**
> While it is appreciated, the rework will first be done locally. Keep an eye out for when the first build is available for testing as it is then when contributions will be accepted to this update.

**What will happen to TempFly_ER after rework is completed?**
> This repository will become a read-only Public Archive and all Releases will be removed. Using the updates in this repository after RevFly is completed would require you to manually compile TempFly_Rev with Maven from your own local development enviroment.

**I have a question that's not listed here.**
> Head over to [my Discord server](https://discord.gg/jP2uyYzbCA) to ask it. (Make sure you select "Plugins" from Onboarding questions or "Channels & Roles" to see TempFly channels).
> I am also building a more development-oriented Discord server which will be open to the public once RevFly is completed.
